# Sprints

Diese Sprintplanung uebersetzt die Roadmap in eine praktische Bau-Reihenfolge.

## Sprintziel pro Phase

- Sprint 1: Szene und Brett-Interaktion
- Sprint 2: Regeln und Zugfluss
- Sprint 3: Platzhalterfiguren und komplette Spiellogik
- Sprint 4: Blender-Assets integrieren
- Sprint 5: UX, Animationen, Sounds
- Sprint 6: Bot oder Multiplayer

## Pflege

Wenn Prioritaeten oder Deliverables verschoben werden, muessen `ROADMAP.md` und diese Sprintdokumente synchronisiert werden.
